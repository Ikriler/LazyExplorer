Expressions

Last Updated: October 28, 2017

When simple workflows don't provide the flexibility to perform the more advanced tasks, the Expressions feature may provide a higher degree of control. Expressions can contain variables which can be set and links can use expression logic to determine the workflow direction.


Constructing an Expression

Expressions can either be constructed either in order to have the result assigned to a variable (Set Variable, CSV Import) or in order to test whether the expression is true (Test Expression).

The above images show how a variable can be set, tested against and used as a field value.

  • We initially use the Set Variable event with the expression "1+1" which literally sets the variable "pagenumber" to the value of 2.
  • We can then use the Test Expression link type to ensure the "pagenumber" variable is at least 1 (Test Expressions must always be a true/false query).
  • Finally, we use the "pagenumber" variable as the value for an input by prepending the '$' sign to the variable name.

You cannot evaluate queries outside of the Set Variable and Text Expression types. You can only use preset variables by themselves.

If you need to use the literal '$' symbol as the first character of a value, prepend an additional dollar sign to ensure it's not used as a variable. For example, to set a value to a price, enter it like $$1.23.

Reserved Variables

The system reserves the use of any variables starting with an underscore. You cannot set these variables directly, however you may use these in your own workflows. Values are regenerated immediately before evaluation.

Variable Description
_OS The operating system being used ("windows", "mac", "linux" or "unknown")
_BROWSER The browser being used ("chrome", "firefox", "opera" or "unknown")
_WINDOW_WIDTH The width (in pixels) of the simulation window
_WINDOW_HEIGHT The height (in pixels) of the simulation window
_FINDTEXT_X The X-coordinate result from a Find Text event
_FINDTEXT_Y The Y-coordinate result from a Find Text event
_FINDTEXT_SELECTOR The CSS Selector of the element found with a Find Text event, if found
_FINDIMAGE_X The X-coordinate from the center of the result from a Find Image event
_FINDIMAGE_Y The Y-coordinate from the center of the result from a Find Image event
_FINDIMAGE_X1 The X-coordinate from the left-edge of the result from a Find Image event
_FINDIMAGE_Y1 The Y-coordinate from the top-edge of the result from a Find Image event
_FINDIMAGE_X2 The X-coordinate from the right-edge of the result from a Find Image event
_FINDIMAGE_Y2 The Y-coordinate from the bottom-edge of the result from a Find Image event

Syntax

You can group together operations and even evaluate functions within an expression, similar to the way a mathematical formula works. Below is a table of the available operators, in the order they are processed:

Operator(s) Description
"text" Enclose plain text in double quotes
(sub-expression) Grouping (order of operations)
f() Function Calls (see below for full list)
x! Factorial
x^y Exponentiation (to the power of)
* , / , % Multiplication, Division and Modulus (remainder)
+ , - Plus and Minus (plus can also be used to join strings)
== , != , >= , <= , > , < Equals, Not Equals etc.
and Logical and (used in Text Expressions to join queries)
or Logical or (used in Text Expressions to join queries)

And the following is a table of the available functions:

Function Description
if(c,a,b) If c is true, the result is a, otherwise the result is b
random(n) Get a random number in the range 0 - n (If n is zero, or not provided, it defaults to 1)
min(a,b,…) Get the smallest (minimum) number in the list
max(a,b,…) Get the largest (maximum) number in the list
length(x) The amount of text characters in x
ceil(n) n rounded up to the nearest whole number
floor(n) n rounded down to the nearest whole number
contains(a,b) Whether the text a contains the text b
startswith(a,b) Whether the text a starts with the text b
lookup(a) Get the value of the variable named a
